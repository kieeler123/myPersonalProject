📘 1. HTML 관련 질문
Q1. セマンティック HTML とは何ですか？

(시맨틱 HTML이 무엇인가요?)

A. 모범답변
HTML 要素に「意味」を持たせて記述する方法です。
検索エンジンやスクリーンリーダーが正しく内容を理解でき、アクセシビリティや SEO の向上につながります。
React でも適切なタグ選びは重要だと考えています。

팁: React 개발자로서 “JSX에서도 중요하다” 멘트 넣으면 가산점.

Q2. <div> と <section> の違いを説明できますか？
A. 모범답변

<div>は意味を持たない汎用コンテナで、
<section>は「テーマを持った内容のまとまり」を示す意味的な要素です。
見出しが付けられるような単位では<section>を使用します。

팁: 일본 기업은 div 乱用(남발)을 싫어함 → 이 차이를 명확히 말하면 Good.

Q3. アクセシビリティを意識している点はありますか？
A. 모범답변
alt 属性の適切な設定、フォームラベルの関連付け、
セマンティックタグの使用、キーボード操作への配慮などを意識しています。

🎨 2. CSS 관련 질문
Q1. Flexbox と Grid の違いを説明してください。
A. 모범답변
Flexbox は 1 次元（横か縦）、Grid は 2 次元（行と列）のレイアウトに適しています。
要素の整列を重視する場合は Flexbox、全体のレイアウト設計は Grid を使っています。

Q2. position: absolute の基準点はどこですか？
A. 모범답변
一番近い position: relative が設定された親要素が基準になります。
設定されていない場合はビューポートが基準になります。

Q3. レスポンシブデザインでは何を意識していますか？
A. 모범답변
モバイルファーストで設計し、ブレークポイントごとに
余白・フォントサイズ・レイアウトの最適化を行います。
また、flex や grid を使って柔軟に配置できるようにしています。

🧠 3. JavaScript 관련 질문
Q1. JavaScript のスコープには何がありますか？
A. 모범답변
グローバルスコープ、関数スコープ、ブロックスコープの三つがあります。
let と const はブロックスコープ、var は関数スコープです。

Q2. 非同期処理について説明してください。
A. 모범답변
処理の完了を待たずに次に進む仕組みです。
Promise、async/await を利用して可読性を保ちながら
非同期処理を扱うようにしています。

Q3. イベントバブリングとは何ですか？
A. 모범답변
子要素で発生したイベントが親要素へ伝播していく仕組みです。
必要に応じて stopPropagation を使って制御します。

⚛️ 1. React 질문 & 모범 답변
Q1. React のコンポーネントとは何ですか？
A. 모범답변
UI を部品として分割し、再利用できるようにしたものです。
単一責任を意識して、小さく保つようにしています。

Q2. Props と State の違いは？
A. 모범답변
props は親から渡される値で読み取り専用、state はコンポーネント内部で管理する変更可能な状態です。

Q3. useEffect の役割を説明してください。
A. 모범답변
副作用処理（データ取得、イベント登録など）を行うためのフックです。
依存配列を指定することで実行タイミングを制御できます。

Q4. レンダリング最適化では何を意識していますか？
A. 모범답변
React.memo、useMemo、useCallback を状況に応じて使い、不要な再レンダリングを避けるようにしています。

Q5. カスタムフックを使う理由は？
A. 모범답변
ロジックを UI から切り離して再利用性を高めるためです。

Q6. React でフォームを扱う際のポイントは？
A. 모범답변
制御コンポーネントで value と onChange を管理し、一元的に状態を扱うようにしています。

🟫 4. DB / API / Express 관련 질문

(프론트엔드 포지션에서도 협업을 위해 자주 나옵니다.)

Q1. REST API とは何ですか？
A. 모범답변
HTTP メソッド（GET/POST/PUT/DELETE）を用いてリソースを操作する API 設計のスタイルです。

Q2. HTTP ステータスコードでよく使うものは？
A. 모범답변
200（成功）、400（Bad Request）、401（認証エラー）、404（Not Found）、500（サーバーエラー）です。

Q3. Cookie と LocalStorage の違いは？
A. 모범답변
Cookie は認証情報などをサーバーと送受信するのに使われ、
LocalStorage はクライアント側だけで保持する長期保存用データです。

Q4. Express を使った経験がありますか？

(경험이 없어도 OK)

A. 모범답변
簡単な API サーバーの構築経験があります。
ルーティング、ミドルウェア、レスポンス処理の基本を理解しています。

경험 없으면 이렇게 말하면 됨:

Express 自体の経験はまだ浅いですが、
REST API の基本的な仕組みは理解していますので、
必要であればすぐに習得できます。

Q5. DB はどの程度理解していますか？
A. 모범답변
フロントエンドとして使用する範囲で、
テーブル構造、主キー/外部キー、SQL の基礎を理解しています。
