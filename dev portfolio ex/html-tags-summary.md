🧱 1. 문서 구조 & 메타

<!DOCTYPE html>

HTML5 문서임을 브라우저에게 알려 돔 파싱 방식을 표준 모드로 설정

없으면 브라우저가 “호환 모드(quirks mode)”로 동작해 CSS 레이아웃이 꼬임

팁: 모든 HTML 파일의 0번 줄 고정값

<html>

문서 전체를 감싸는 루트 요소

lang="ko" 같은 언어 설정이 접근성에 매우 중요

팁: 화면 읽기 프로그램이 페이지 언어를 여기서 판단합니다.

<head>

문서의 “보이지 않는 설정 영역”

SEO, 메타데이터, 스타일, 스크립트, 외부 리소스 모두 head에 들어감

<meta>

문자 인코딩(UTF-8), SEO 설명(description), 뷰포트 설정

특히 viewport는 모바일 대응의 핵심

<title>

검색엔진이 가장 먼저 크롤링하는 정보

탭 제목이자, 북마크 표시 이름

<style>

문서 내 CSS 삽입

“학습용” 또는 “컴포넌트 스코프” 스타일 용도

실무에서는 주로 외부 CSS(link)를 사용

팁: 스타일이 커질수록 style 태그는 유지보수에 불리함

<body>

실제 화면에 보이는 모든 콘텐츠

이 안의 요소는 브라우저가 렌더링하며 사용자와 직접 상호작용 가능

🧩 2. 시맨틱 구조(Structural HTML)

이 영역은 접근성, SEO, 유지보수성을 위해 매우 중요합니다.

<header>

페이지나 섹션의 “소개 영역”

제목, 로고, 메뉴 등이 위치

검색엔진이 “이 부분이 어떤 콘텐츠의 시작인지” 이해하는데 도움

<nav>

내비게이션(메뉴)의 공식 영역

검색엔진이 “이 링크 목록은 메뉴다”라고 파악

스크린리더는 nav를 만나면 “내비게이션 영역”이라고 안내함

<main>

문서의 핵심 콘텐츠

페이지 내 단 1개만 존재해야 함

보조기술이 main을 빠르게 탐색하여 핵심 내용을 찾을 수 있음

<section>

문서의 큰 주제별 묶음

section 자체는 “항목의 구분”이 되지만 독립 콘텐츠는 아님

팁: section 안에는 반드시 제목(h계열)이 포함되어야 시맨틱 의미가 완성됩니다.

<article>

독립적으로 존재해도 의미가 유지되는 콘텐츠

블로그 글 / 뉴스 / 카드 / 포트폴리오 개별 작품 등

팁: article을 잘 쓰면 구글 검색 노출이 좋아짐

<footer>

페이지나 섹션의 바닥 영역

저작권, 연락처, 관련 링크 등을 담음

✏️ 3. 텍스트 관련 태그
<h1>~<h3>

SEO, 접근성 모두에서 중요

h1은 페이지의 최상위 제목, h2는 큰 섹션, h3는 subsection

<p>

문단(paragraph)

줄바꿈 용으로 <br> 남발하면 잘못된 구조가 됨

<strong>

의미 강조 (중요하다는 표현)

스크린리더는 “강조됨”이라고 읽음

검색엔진도 strong을 의미 차원에서 해석

<em>

문맥적 강조 (기울임)

강한 의미보다는 “읽을 때 톤 변화” 같은 뉘앙스를 전달

<mark>

의미 강조 + 하이라이트

검색결과 표시 같은 곳에 사용하면 직관적임

<code>

코드, 명령어, 함수 이름 같은 프로그래밍 요소 표시

monospace 폰트로 렌더링

<time>

날짜/시간을 기계적으로 이해 가능

예: <time datetime="2024-01-20">2024년 1월 20일</time>
→ 검색엔진·캘린더 앱이 정보를 정확하게 읽을 수 있음

<abbr>

약어를 명확하게 설명

마우스 올리면 툴팁 표시(<abbr title="…">)

접근성에서 중요한 단어 설명 기능

🎨 4. 이미지 & 미디어
<figure>

이미지 또는 미디어 + 설명을 묶는 컨테이너

콘텐츠와 설명(캡션)은 하나의 논리적 묶음이 됨

<figcaption>

figure의 설명 부분

SEO적으로 이미지의 의미를 더욱 강화함

<img>

단일 이미지 표시

alt는 필수 (접근성, SEO, 이미지 로딩 실패 대비)

<video>

자체적으로 제공하는 영상 재생기

controls, autoplay, loop 등 HTML 속성으로 동작 제어

외부 서비스(유튜브)는 video 불가 → iframe 사용

<source>

video, audio, picture에서 “서로 다른 포맷/조건”을 제공

브라우저는 지원 가능한 파일을 자동 선택

<iframe>

외부 사이트를 “창문처럼” 심는 태그

지도, 유튜브, 외부 위젯 삽입 가능

보안 속성(sandbox, allow 등) 중요

📊 5. 표 태그
<table>

정형 데이터를 구조화해 표현

그냥 div로 스타일링한 테이블보다 접근성이 훨씬 좋음

<thead> / <tbody> / <tfoot>

구조를 명확히 구분

스크린리더는 thead의 내용을 “헤더”로 읽음

레이아웃, sticky header 등 스타일링 시 매우 유리함

<tr>

표의 행(Row)

<th>

제목 셀

기본적으로 bold + 중앙 정렬

scope 속성을 줘서 관계 명확하게 할 수 있음

<td>

일반 데이터 셀

📮 6. 폼(Form) 관련 태그
<form>

사용자 입력을 서버로 전송하기 위한 구조

method(action) 옵션을 통해 전송 방식 지정 가능

<label>

입력 요소 설명

for="id" 속성으로 input과 연결
→ 사용자가 라벨을 클릭해도 input에 포커스가 감

핵심 UX 포인트: 접근성에서 매우 중요

<input>

텍스트, 이메일, 비밀번호, 날짜 등 “다목적 단일 입력"

type 값에 따라 동작이 완전히 달라짐

<select>

드롭다운(목록 선택)

여러 개 중 하나를 고르게 하는 경우 이상적

<option>

select 안의 선택지

<textarea>

여러 줄의 입력 텍스트

블로그 글 작성, 긴 메모 등

<button>

제출 또는 액션 실행

type="submit", type="button" 등으로 역할 조정

📌 총평 – 이 예제 페이지의 설계 의미

이 예제는 단순한 HTML 페이지처럼 보이지만 사실:

시맨틱 구조를 전부 포함

텍스트 의미 태그를 다양하게 활용

미디어(video/iframe) 비교 가능

테이블/리스트 사용법 모두 포함

폼까지 완비

접근성·SEO 관점에서 바람직한 구조

즉, HTML 전체의 80% 기능이 자연스럽게 들어 있는 학습용 기본기 템플릿이에요.
